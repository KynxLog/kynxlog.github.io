微信小程序开发体验

以前一直是知道微信小程序是由前端的三个语言组合开发出来的，但是现在会变得如此简单，省去很多学习前端时不需要的后端知识。我做的是一个手机硬件数据库类型的小程序，目前已经搭建了一个雏形出来，但是还有很多的小功能需要去完善。 在这些天的开发之中学习到了很多新鲜的东西，就包括前端代码是怎么操作后端数据库（微信云开发的数据库），以及很多比float更好的布局方式，以及垂直向下滚动还需要代码…… HTML / CSS — 布局 我一直非常喜欢有规矩的布局方式，就类似于网格样式的卡片布局，所以之前一直在研究怎么漂浮可以做到，但是一直是没有成果的，所以索性问问AI，看看有什么比较好的方法。 AI这个时候就给我推荐了flex布局，但是flex布局是单项的，只能但横向或者是单竖向排列： <div class="container"> <img src="*"> <text>Hello</text> </div> .container { display: flex; flex-direction: column; /* 这个是纵向布局，也就是两个元素竖向排列，column-reverse就是将首尾颠倒摆放 */ flex-direction: row; /* 这个是横向布局，也就是两个元素横向摆放，row-reverse就是将上下元素颠倒摆放 */ gap: 6px; /* 添加两个元素之间的间距，单纵向或者是横向 */ } 但是还是不容易实现类似网格的样式，要是列数不一样就会显示成居中的样式，还是非常难受的，类似于这样： 所以，我开始寻找一个更好的布局方式，晚上偶然间刷哔哩哔哩的时候看到了 这个视频 之后学会了使用Grid布局，这个布局我后来是将其混合进flex布局里面，个人感觉看上去还不错，这里放一段小程序源码片段： <view class="items" wx:for="{{products}}" wx:key="_id" bindtap="goToDetail" data-id="{{item._id}}">…
